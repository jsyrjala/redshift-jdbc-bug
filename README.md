# Bug in Redshift JDBC driver

redshift-jdbc42-no-awssdk contains buggy date/timestamp handling.

This throws StackOverflowException in newer redshift JDBC driver versions: 
```java
new PGTimestamp(1).toString()
```

Affected redshift JDBC versions:
- 1.2.43.1067
- 1.2.41.1065

Version 1.2.37.1061 and earlier seem to work correctly:

Versions tested to work:
- 1.2.37.1061
- 1.2.34.1058
- 1.2.16.1027
- 1.2.15.1025
- 1.2.10.1009

Tested with JDK 11
 
## Running tests

- change version number for redshift-jdbc42-no-awssdk in `pom.xml`
- run tests
```
mvn clean package
```
- depending on the version number either the tests pass or fail with StackOverflowException.

## Affected software
Among other things IntelliJ IDEA Database browser fails to show timestamps correctly
when doing this query in Redshift using affected JDBC versions and throws StackowerflowException instead.
```
select now();
```
